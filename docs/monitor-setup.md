# Monitor Agent Setup

PURA's daily digest agent (Phase 5.5) is live in code but needs one external dependency before the emails go out: Resend.

## What you need to do (10 minutes)

### 1. Create a Resend account

Go to **resend.com** → sign up with viveapr@gmail.com → free plan (3,000 emails/month, more than enough).

### 2. Add the purasignal.com domain

In the Resend dashboard → **Domains** → **Add Domain** → enter `purasignal.com`.

Resend will give you 3 DNS records to add (typically 2 TXT + 1 CNAME). Add them in **Cloudflare** → purasignal.com DNS. Verification takes 1–5 minutes.

### 3. Create an API key

Resend → **API Keys** → **Create API Key** → name it `pura-digest` → copy the key.

### 4. Add to Vercel env vars

Vercel Dashboard → purasignal project → **Settings** → **Environment Variables**:

```
RESEND_API_KEY = re_xxxxxxxx...   (from step 3)
```

Set for: Production + Preview + Development.

Redeploy (or just wait for the next deploy) for the env var to take effect.

## How it works after setup

| Cron | UTC | CT | What it does |
|------|-----|----|--------------|
| `monitor-retry` | 14:00 | 9:00 AM | Retries briefings that failed in morning-send |
| `monitor-digest` | 03:00 | 10:00 PM | Gathers 24h data, runs one last retry, sends email |

The digest email comes from `digest@purasignal.com` → `viveapr@gmail.com`.

---

## What the digest email contains

- **Status banner** — green if no anomalies, red with bullet list if issues detected
- **Check-ins today + last 7 days**
- **Morning run table** — per clinic: briefing generated (✓/✗), cron log present (✓/✗)
- **Auto-retry results** — if any clinic had a retry attempt, outcome listed
- **Error events** — last 24h access_log entries matching `fail` or `error` pattern

---

## Anomaly detection

The digest flags:
- Any clinic that has no briefing generated by 10pm
- Zero check-ins across all clinics on a day with active patients
- 5+ error-pattern events in 24h (possible elevated failure rate)
- Morning cron not logged for any clinic

## Testing without waiting for 10pm

From your Mac terminal (after `gh auth login`):

```bash
curl -X POST https://purasignal.com/api/cron/monitor-digest \
  -H "Authorization: Bearer $CRON_SECRET"
```

Or trigger from Vercel Dashboard → Functions → /api/cron/monitor-digest → Test.

---

## Vercel runtime logs

The digest and retry routes log structured JSON to Vercel's runtime log on every run. Visible at:
Vercel → purasignal → **Logs** → filter by `/api/cron/monitor-*`
